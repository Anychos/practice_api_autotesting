# Online Shop API Automation Testing

Это проект по автоматизации тестирования API онлайн магазина. Он был создан с целью практической отработки навыков, полученных после прохождения курса по автоматизированному тестированию API.

---

## О проекте

Проект нацелен на автоматизацию тестирования для базовых CRUD операций (создание, получение, обновление, удаление) для онлайн магазина с различными сущностями:
- **Authentication** – аутентификация пользователей с разными ролями (admin, user)
- **Users** – управление пользователями
- **Products** – управление товарами
- **Carts** – взаимодействие с корзиной покупок
- **Orders** – создание и обработка заказов

---

## Технологии

Проект использует следующие технологии:
- **Python 3.13**
- **Pytest** — фреймворк для написания тестов и их управления (фикстуры, плагины, маркеры)
- **HTTPX** — современная HTTP библиотека для взаимодействия с API
- **Allure** — инструмент для генерации отчетов (шаги, теги, критичность)
- **Faker** — библиотека для генерации тестовых данных
- **Pydantic** — инструмент для валидации схем запросов и ответов
- **Swagger coverage tool** — инструмент для измерения покрытия API тестами
- **PostgreSQL** — база данных для их хранения и управления

---

## Особенности реализации

- **Объектно-Ориентированное Программирование (ООП):** проект реализован с использованием принципов ООП, что позволяет избегать дублирования кода и улучшает его читаемость и масштабируемость.
- **Изолированные тесты:** все тесты независимы друг от друга и могут быть выполнены в любом порядке.
- **Тестирование позитивных и негативных сценариев:** написаны тесты как для корректных, так и некорректных запросов.
- **Интеграция с базой данных:** тесты взаимодействуют с реальной локальной базой данных. В среде CI используется service PostgreSQL, что позволяет поднимать временную базу данных во время прогона тестов.
