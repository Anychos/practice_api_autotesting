# Online Shop API Automation Testing

Этот проект представляет собой реализацию автоматизированных тестов для API онлайн магазина. Он был создан с целью практической отработки навыков, полученных после прохождения курса по автоматизированному тестированию API.

---

## О проекте

Проект нацелен на автоматизацию тестирования для базовых CRUD операций (создание, получение, обновление, удаление) для онлайн магазина с различными сущностями:
- **Authentication** – аутентификация пользователей с разными ролями (admin, user)
- **Users** – управление пользователями
- **Products** – управление товарами
- **Carts** – взаимодействие с корзиной покупок
- **Orders** – создание и обработка заказов

Тесты проверяют как позитивные, так и негативные сценарии для этих операций.

---

## Технологии

Проект использует следующие технологии:
- **Python 3.13**
- **Pytest** — фреймворк для написания тестов (фикстуры, плагины, маркеры)
- **HTTPX** — HTTP клиент для взаимодействия с API
- **Allure** — для генерации отчетов (шаги, теги, критичность, теги)
- **Faker** — для генерации тестовых данных
- **Pydantic** — для валидации схем запросов и ответов
- **Swagger** — для генерации документации по уровню покрытия API тестами
- **PostgreSQL** — база данных для хранения данных

---

## Особенности реализации

- **Объектно-Ориентированное Программирование (ООП):** проект реализован с использованием принципов ООП, включая инкапсуляцию и наследование.
- **Изолированные тесты:** все тесты независимы и могут быть выполнены в любом порядке.
- **Тестирование позитивных и негативных сценариев:** написаны тесты как для корректных, так и для некорректных запросов.
- **Интеграция с базой данных:** тесты взаимодействуют с реальной базой данных.
